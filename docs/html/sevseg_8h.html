Detailed Description

Raspberry Pi Pico C Library for a 4-Digit 7-Segment Display.

This library supports a 4-Digit 7-Segment display with the Raspberry Pi Pico in C. It contains functions to allow for displaying many common symbols and custom configurations of segments on any digit in the display using a polling refresh function.

void InitSevseg ( Sevseg *  sevseg)

Initialize the digit mappings based on display's pins.

The 7-segment display requires dedicated pins for controlling each digit column and each segment within a digit column. This function takes the provided wiring configuration for all dedicated pins and calculates the necessary pin mapping to display every common 7-segment symbol. These calculations are stored at each Sevseg instance in digit_map to ensure faster lookup of symbol values during runtime.

[in]sevsegA pointer to a 7-Segment Object

A function to manually configure a single 7-segment digit.

This function takes a referenced 7-segment object, a manually created symbol out of a one-hot encoded segments byte, and the desired digit position to write to. The passed digit position in digit_codes is updated with the passed segments byte. The segments byte follows the format of 0bPGFEDCBA where "P" represents controlling the decimal-point.

[in]sevsegA pointer to a 7-Segment Object

A function to encode a number to the 7-segment format.

This function takes a referenced 7-segment object, an 16-bit unsigned integer value, a hex indicating flag, and the number of decimal places for the desired number. The integer value is translated into coresponding numeric symbols from the object's digit_map and stores it in digit_codes. Whether the passed value is interpreted as a decimal or hex value is deduced from the hex flag and which digit column to place the decimal is controlled by decPlaces.

Parameters
[in]sevsegA pointer to a 7-segment object
[in]valueA 16-bit unsigned integer to write across the 7-segment display
[in]hexA boolean to indicate whether the value is coded in hex or decimal
[in]decPlacesA byte to indicate how many decimal places should be displayed
